#.vscdoe 에디터 초기 셋팅 방법

 1.vscode/extensions.json 파일 : eslint, prettier 필수적은 플러그인을 자동으로 설치해주는 설정 파일입니다.
  -온라인 환경에서만 자동 설치가 됩니다.
  -오픈라인이 경우 아래 그림처럼 docs/vscode-plugin 폴더에 있는 각 파일의 context-menu 중 [확장 VSIX 설치] 메뉴를 통해 오프라인 설치를 진행합니다.
 2.vscode/settings.json 파일 : vscode 에디터 셋팅을 통일시키기 위한 에디터 설정 파일입니다.

 3.소스를 다운로드 받고 최초에 docs 폴더 있는 .env.development.local 파일을 env 폴더로 복사를 합니다.
  : .env.development.local 파일은 git push 대상이 아니고 로컬 개발시 해당 환경 파일이 적용되기 위함입니다.
  : 해당 환경 파일에 로컬 서버 URL 정보들을 재정의해서 사용하면 됩니다.


#.환경 변수 가이드

 1.env 폴더 파일 설명
  -.env : "운영" 환경 기준 환경변수들 정의되어있습니다
  -.env.development : "개발" 환경 기준 환경변수
  -.env.stage : "스테이징" 환경 기준 환경변수
  -.env.publish : 퍼블리싱 APP을 실행시키기 위한 환경변수

 2.환경 변수 속성
  -ENABLE_PROXY_LOG : 로컬환경에서 vite 사용시 local 서버 호출시 CORS 적용을 우회하기 위해서 proxy 방식으로 호출되는데 이때 proxy 서버 requet/response 로그를 터미널에 표기하기 위한 설정(로컬 개발환겨에서만 사용됩니다.)
  -VITE_LOCAL_DEVELOP : 로컬 개발시 특정 동작을 위하기 위한 설정 정보입니다.(예를들면 SSO 로그인 같은 부분은 로컬환경에서 다르게 동작시키기 위한)
  -VITE_MAIN_PAGE : 고정값입니다. 변경 불필요
  -VITE_API_PREFIX : 서버 API URI prefix 정보입니다. 수정시 모든 환경과 동일하니 .env 파일만 수정하면 됩니다.
  -VITE_API_URL : 현재 로컬개발시 vite proxy를 사용하므로 빈값으로 고정으로 설정됩니다. 
  -VITE_LOCAL_API_URL : 로컬개발시 axios 호출시 proxy 방식으로 서버 호출이 되므로 해당 URL을 정의해주면 됩니다.(빌드환경에서는 사용되지 않는 변수입니다.)
  -VITE_SERVER_CONTEXT_PATH : servlet context 경로로 예를들어 배포된 환경이 war 기반인데 root 경로가 아니고 survey context-path 기준으로 배포가 될 경우 해당 경로를 정의해주면 됩니다.

#.기본 아키텍트

 1.빌드 구조
  -프로젝트 구성상 SPA에 적합한 프로젝트입니다. nextjs는 SSR 기반의 프로젝트가 아니면 러닝커브가 높고 npm 버전별도 동작이 다르므로 빌드환경을 vite로 결정하였습니다.
  -빌드와 로컬실행등 실행환경에 대한 정보를 다르게 가져가야 하는 경우 환경파일 기준으로 분류되고 env 폴더에 각 환경별로 환경파일을 등록하여 관리합니다.
  -빌드 완료시 dist 폴더에 결과물이 추출되는데 해당 결과물을 nginx 폴더에 복사하면 됩니다.

 2.주요 프론트 라이브러리 버전 : 각 라이브러리와 호환성을 react 버전을 18.x 채택하였습니다. 나머지 주요 라이브러리는 최신 버전으로 반영되었습니다. 개발시 시간이 지나면 antd, aggrid, zustand, react-router-dom 라이브러리 마이너/패치 버전을 업로드할 필요가 존재할 수 있습니다.
  -react : 18.x
  -antd : 6.x
  -zustand: 5.x
  -aggrid : 35.x
  -react-router-dom : 7.x

 3.주요 개발 패턴
  -상태 관리를 useState가 아닌 zustand store로 대신합니다.
  -테이블 및 공통 input 컴포넌트는 공통 컴포넌트를 사용합니다.
  -폼의 유효성 체크는 yup 라이브러리를 사용합니다.
  -불변 라이브러리는 immer, user-immer를 사용하면 zustand와 조합하여 사용합니다.

 4.공통 컴포넌트 : 상세 사용법은 각 기능별 가이드 URL 참고
  -src\components\common\AppButton.tsx : 공통 버튼

#.실행 스크립트

 1.npm run dev
  -로컬 개발

 2.npm run dev:publish
  -퍼블리싱

 3.npm run build
  -빌드 수행 dist 폴더에 결과물 추출

 4.npm run build:pass
 -동일하게 빌드를 수행하지만 문법 오류 체크하지 않고 실행



#.주요 파일/폴더 설명

 1./(ROOT 폴더)
  -.prettierrc : prettier 설정 수정하고 싶을 경우 수정
  -eslint.config.js : eslint 설정 수정하고 싶을 경우 수정
  -vite.config.ts : local proxy 서버 설정 및 기본 빌드 설정
  -index.html : 최종 빌드되는 template html 파일

 2.docs
  -nginx.conf : nginx proxy 서버 설정 파일(각각 아래의 정보를 replace)
   : http://localhost:8080 => 서버 도메인
   : http://localhost => 프론트 도메인
  -.env.development.local : 로컬 개발 시작시 해당 파일을 env 파일에 복사

 3.env : 환경변수 파일 폴더

 4.public : 정적 리소스 파일 모음 폴더(빌드시 해당 폴더의 resource도 같이 반영됩니다.)

 5.src/components
  -common : 공통 컴포넌트
  -layout : 레이아웃 관련 컴포넌트
  -modal : 모달

 6.src/config : 설정, 상수 파일 모음 폴더
  -Code.ts : 프론트 전용 코드 파일
  -CmmonConstant.ts : 상수
  -Config.ts : 전역 설정 객체
  -ModalType.ts : 모달 유형 상수

 7.src/data : 실제 적용되는 메뉴 데이터, 테이블 예제 데이터 폴더

 8.src/guide : 개발 가이드 컴포넌트 모음 폴더

 9.src/hooks : 공통 hook 폴더

 10.src/page : 메뉴에 매핑되어있는 페이지 모음 폴더

 11.src/publish : 퍼블리싱 작업 파일들 모음 폴더

 12.src/resources : scss, 이미지, 폰트 파일

 13.src/routes : 라우터 설정

 14.services : 공통 유틸 서비스

 15.src/store : zustand store 파일 모음
  -common : 공통 store 폴더
  -slice : 공통 기능 모음 slice(form, list) 폴더
  -useAppStore.ts : 전역 데이터 stroe
  -useUiStore.ts : 전역 UI store

 16.src/utils : 유틸성 함수

주요 폴더 및 파일 구조

📂 Root Directories

/ (Root): 프로젝트 설정 파일

.prettierrc / eslint.config.js: 코드 스타일 및 린트 설정

vite.config.ts: Vite 빌드 및 로컬 Proxy 서버 설정

index.html: 앱의 메인 엔트리 템플릿 파일

docs/: 문서 및 서버 설정 리소스

nginx.conf: Nginx Proxy 서버 설정 (도메인 Replace 정보 포함)

.env.development.local: 로컬 개발용 환경 변수 템플릿

env/: 각 환경별(.env, .development, .stage, .publish) 설정 파일

public/: 빌드 시 변환 없이 복사되는 정적 리소스 (파비콘 등)

📂 Source Code (src/)

프로젝트의 핵심 소스코드는 src 폴더 내에 기능별로 구조화되어 있습니다.

components/: 모든 화면 구성 요소

common/: 버튼, 입력창 등 공통 UI 컴포넌트

layout/: 헤더, 사이드바 등 레이아웃 컴포넌트

modal/: 팝업 및 모달 관련 컴포넌트

config/: 전역 설정 및 상수

Code.ts (프론트 전용 코드), CommonConstant.ts (상수), Config.ts (설정 객체), ModalType.ts

pages/: 라우터와 매핑되는 메뉴별 메인 페이지 (예: Login.tsx, Home.tsx)

store/: Zustand 기반 상태 관리

common/: 공통 스토어

slice/: 기능별 조각(form, list 등) 관리

useAppStore.ts (전역 데이터), useUiStore.ts (UI 제어)

data/: 실제 바인딩되는 메뉴 데이터 및 테이블 예제 데이터

routes/: 전체 앱의 페이지 라우팅 경로 설정

services/: 공통 비즈니스 로직 및 API 호출 서비스

resources/: 스타일 파일(SCSS), 이미지, 웹 폰트 등 에셋

utils/: 프로젝트 전반에서 재사용되는 헬퍼 함수

hooks/: 공통 React Custom Hooks

guide/: 개발자용 컴포넌트 사용 가이드 페이지

publish/: 퍼블리싱 작업 결과물 관리
